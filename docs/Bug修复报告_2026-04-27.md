## Bug 1: gopeed.db 路径不一致导致残留任务阻塞新下载

### 错误点

- `core/api/app.py:34` 硬编码了 gopeed.db 路径为 `weixin_exe/gopeed.db`
- `config/settings.py` 的 `gopeed_db_path` 属性在打包环境下返回 `lib/gopeed.db`
- 两者路径不一致，导致软件重启时删除了错误的数据库文件（或没删掉应该删的）

### 原因

1. **路径硬编码**：`_clear_go_tasks_safe()` 函数直接使用硬编码路径，而不是通过 `settings.gopeed_db_path` 获取
2. **打包/源码环境差异**：
   - 源码环境：Go 后端在 `weixin_exe/` 目录，gopeed.db 也在该目录
   - 打包环境：Go 后端在 `lib/` 目录，gopeed.db 也在 `lib/` 目录
3. **残留任务阻塞**：Go 重启后，Python 本地 DB 中残留的 `running`/`pending` 任务在 Go 端已不存在，但 `create_download_task()` 检测到活跃任务存在就返回旧 task_id，导致新下载被静默阻塞

### 影响范围

- 用户添加作者后，部分视频无法下载（无任何报错提示）
- 需要手动删除作者重新添加才能恢复

### 修改文件

| 文件 | 修改内容 |
|------|----------|
| `config/settings.py:285-289` | `gopeed_db_path` 属性区分打包/源码环境返回正确路径 |
| `core/api/app.py:34-41` | `_clear_go_tasks_safe()` 使用 `settings.gopeed_db_path` 获取路径 |
| `core/api/app.py:93-97` | `_auto_start_go_backend()` 启动后调用 `cleanup_stale_tasks()` 清理残留任务 |
| `core/service/task.py:466-503` | 新增 `cleanup_stale_tasks()` 静态方法，清理 Go 端不存在的残留任务 |

## Bug 3: 直播回放时长显示错误（显示几秒而非几小时）

### 错误点

- 直播回放时长应显示 3-4 小时，实际只显示 6-20 秒
- 根因：微信接口 `videoPlayLen` 字段值不准确，真实时长在 `spec[].durationMs` 字段中

### 原因

1. **微信接口问题**：`replay/list` 接口返回的 `videoPlayLen` 字段值不正确（只有几秒），但文件大小是正确的（几 GB）
2. **字段选择错误**：我们使用 `videoPlayLen` 字段获取时长，但直播回放的真正时长在 `spec[0].durationMs` 字段中
3. **数据对比**：
   - `videoPlayLen: 6552ms` = 6.5 秒 ❌
   - `spec[0].durationMs: 6553041ms` = 1.82 小时 ✓
   - `fileSize: 3986 MB` ✓ 符合直播回放特征

### 影响范围

- 所有直播回放视频时长显示错误
- 用户无法直观判断视频实际长度
- 影响用户体验，可能误以为视频损坏

### 修改文件

| 文件 | 修改内容 |
|------|----------|
| `core/service/search.py:429-468` | `_parse_video()` 方法从 `spec[0].durationMs` 获取时长 |
| `core/service/video.py:428-473` | `_parse_video_object()` 方法从 `spec[0].durationMs` 获取时长 |
| `tests/单元测试/bug修复/test_duration_bug.py` | 新增测试用例验证修复 |

## Bug 6: CSV 下载模板按钮点击无响应

### 错误点

- 前端点击"下载模板"按钮没有任何反应
- 浏览器控制台显示请求被拦截

### 原因

1. **路由权限问题**：`/api/inputer/csv/template` 端点注册在带 `require_wechat` 依赖的主路由上，未连接微信时返回 503
2. **下载方式不兼容**：pywebview 环境中 `window.open()` 和 `fetch+blob` 均无法触发文件下载
3. **后端路径错误**：`gui.py` 中 `ExposedAPI.download_template()` 调用的路径 `api/authors/import-template` 不存在

| `core/api/routers/inputer.py:65` | `upload-csv` 移入 `public_router`（上传文件不需要微信连接） |
| `core/api/routers/inputer.py:224` | `upload-excel` 移入 `public_router`（上传文件不需要微信连接） |
| `core/api/app.py:116` | 注册 `inputer.public_router` |
| `gui.py:116-118` | 修复 `download_template()` API 路径为 `api/inputer/csv/template` |
| `static/js/component/search.js:151-170` | `downloadCSVTemplate()` 优先调用 `pywebview.api.download_template()`，回调通过 `window._downloadCallback`，浏览器环境降级 fetch+blob |

### 前端代码详情

```javascript
// search.js - 全局下载回调（由 gui.py evaluate_js 调用）
window._downloadCallback = function(result) {
  if (result && result.success) {
    showToast({ type: 'success', title: '下载成功', message: result.message });
  } else {
    showToast({ type: 'error', title: '下载失败', message: result?.message || '未知错误' });
  }
};

// search.js - downloadCSVTemplate() 改动
function downloadCSVTemplate() {
  // pywebview 环境使用原生 API（fetch+blob 在 pywebview 中无法触发下载）
  if (window.pywebview && window.pywebview.api && window.pywebview.api.download_template) {
    // 调用原生 API，结果通过 window._downloadCallback 回调
    window.pywebview.api.download_template();
    return;
  }
  // 浏览器环境使用 fetch + blob 下载
  fetch('/api/inputer/csv/template')
    .then(r => {
      if (!r.ok) throw new Error(`下载模板失败: ${r.status}`);
      return r.blob();
    })
    .then(blob => {
      const url = URL.createObjectURL(blob);
      const a = document.createElement('a');
      a.href = url;
      a.download = 'author_template.csv';
      a.click();
      URL.revokeObjectURL(url);
    })
    .catch(e => {
      console.error('downloadCSVTemplate:', e);
      showToast({ type: 'error', title: '下载模板失败', message: e.message });
    });
}
```

## Bug 8: CSV/Excel 导入不支持 Excel 转换后的日期格式

### 错误点

- 用户用 Excel 打开 CSV 模板后保存，Excel 自动把 `2026-01-01` 转成 `2026/1/1`
- CSV 解析器只认 `YYYY-MM-DD` 格式，报错：`search_value 必须是 YYYY-MM-DD 格式, 实际 '2026/1/1'`

### 原因

1. **日期格式校验过于严格**：`CSVParser` 和 `ExcelParser` 的日期校验只支持 `%Y-%m-%d` 格式
2. **Excel 自动转换**：Excel 打开 CSV 后会自动将日期格式转为 `YYYY/M/D` 或 `YYYY/M/D HH:MM:SS`

### 修改文件

| 文件 | 修改内容 |
|------|----------|
| `core/utils/inputer.py:188-210` | 新增 `_normalize_date()` 静态方法，兼容多种日期格式 |
| `core/utils/inputer.py:162-166` | `CSVParser` 日期校验改用 `_normalize_date()` |
| `core/utils/inputer.py:361-364` | `ExcelParser` 日期校验改用 `_normalize_date()`，移除冗余 DEBUG 日志 |

### 验收条件

```bash

.venv/Scripts/python -c "

for d in tests:
    result = CSVParser._normalize_date(d)
    print(f'  {d!r:35s} -> {result}')
"
# 所有格式都应标准化为 2026-01-01
```

### 效果

- ✅ 支持 `2026-01-01`、`2026/1/1`、`2026/01/01`
- ✅ 支持带时间 `2026-01-01 00:00:00`、`2026/1/1 12:30:45`
- ✅ 支持 ISO 格式 `2026-01-01T10:30:00`
