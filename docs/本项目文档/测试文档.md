## 测试数据库使用规范

### 核心原则

测试必须使用 **内存数据库 (`:memory:`)**，严禁直接操作生产数据库。所有测试数据必须在测试结束后清理。

### 数据库生命周期

每个测试的数据库生命周期如下：

| 阶段 | 说明 |
|------|------|
| 创建 | 每次测试创建新的 `:memory:` 数据库 |
| 注入 | 通过 `monkeypatch` 替换服务模块中的 `db` 引用 |
| 运行 | 测试执行 |
| 清理 | fixture teardown 删除测试数据、关闭数据库 |

### conftest.py 提供的 Fixtures

#### 1. `inject_test_db` (autouse=True)

自动注入测试数据库到服务。每次测试自动创建新数据库，测试结束后关闭。

```python
@pytest.fixture(autouse=True)
def inject_test_db(monkeypatch):
    test_database = Database(":memory:")
    monkeypatch.setattr(db_base, "_db_instance", test_database)
    monkeypatch.setattr(db_module, "db", test_database)
    yield test_database
    test_database.close()
```

#### 2. `db`

内存数据库实例，用于在测试中创建数据：

```python
@pytest.fixture
def sample_author():
    return Author(id="author_001", name="test", ...)
```

### VideoService 测试的特殊处理

**问题**：`VideoService` 在文件顶部 `from core.utils.database import db`，导入时就绑定了旧的 `db` 引用，普通的 `inject_test_db` 无法替换它。

**解决方案**：在测试 fixture 中额外 monkeypatch 服务模块：

```python
@pytest.fixture
def video_service_with_test_db(monkeypatch):
    from core.utils import database as db_module
    import core.utils.database.base as db_base

    test_database = Database(":memory:")

    # 替换全局单例
    monkeypatch.setattr(db_base, "_db_instance", test_database)
    monkeypatch.setattr(db_base, "_db_path", str(test_database.db_path))

    # 替换模块的 db 属性
    monkeypatch.setattr(db_module, "db", test_database)

    # 替换 VideoService 中已导入的 db 引用（关键！）
    import core.service.video as video_module
    monkeypatch.setattr(video_module, "db", test_database)

    yield test_database
    test_database.close()
```

### 注意事项

1. **禁止直接导入生产 db**：严禁 `from core.utils.database import db`，必须通过 fixture 注入
2. **所有服务模块都要 patch**：除了 `db_module.db`，还需 patch 使用了 `db` 的服务模块（如 `VideoService`）
3. **teardown 必须清理**：每个 fixture 的 yield 后必须清理数据
4. **内存数据库无持久化**：`:memory:` 数据库在连接关闭后自动销毁，无需手动删除文件

### Bug 修复记录：Excel 导入绕过 SearchService + 日期过滤字段名错误

- **Bug 1**：`ExcelClient.import_from_excel_to_database()` 绕过 SearchService，直接调用 `wechat_client.get_author_videos()`，导致原始后端数据字段名与 AuthorVideo 模型不匹配
- **Bug 2**：date 模式用 `v.get("create_time")[:10]` 过滤日期，但原始数据字段是 `createtime`（时间戳数字），不是 `create_time`（ISO 字符串），导致 TypeError 崩溃
- **Bug 3**：pages 模式直接传原始 `data` 给 AuthorVideo，所有字段映射错误（title/url/cover_url 全为空字符串）
- **根因**：Excel 导入完全独立实现，没有复用 CSV 导入已验证过的 SearchService 标准化流程
- **修复**：将 `import_from_excel_to_database()` 改为通过 `SearchService` 搜索作者和获取视频，与 CSV 导入保持一致
  - `search_type="date"` → `service.get_author_videos_before_date(source_author_id, search_value)`
  - `search_type="pages"` → `service.get_author_videos(source_author_id, pages=N)`
  - `search_type="all"` → `service.get_author_videos(source_author_id, pages=N)`
- **测试**：`test_excel导入search_type分发与日期过滤.py`（10个验收点，TDD 红绿流程验证）

### Bug 修复记录：腾讯文档导入返回任务描述而非视频数据 + 日期过滤方向反转

- **Bug 1**：`import_author_videos_by_date()` 和 `import_author_videos_by_pages()` 返回 `[{author_name, search_type, search_value}]` 任务描述数据，而非视频对象。所有视频元数据（title, url, cover_url, duration）完全丢失
- **Bug 2**：`import_author_videos_by_date()` 日期过滤方向反转——方法名说"before_date"（日期之前），但实际应该保留"日期到今天"的视频（与 CSV 一致）

- **修复**：将 `import_from_doc_to_database()` 改为通过 `SearchService` 搜索作者和获取视频
  - 与 CSV/Excel 保持一致的标准化流程
  - 删除对 `import_author_videos_by_date/pages` 的依赖
  - 视频字段映射由 `SearchService._parse_video()` 统一处理
- **测试**：`test_腾讯文档导入search_type分发与视频数据完整性.py`（9个验收点，TDD 红绿流程验证）
- **前端测试**：`test_导入面板交互与模板下载.js`（18个验收点——导入面板显示/隐藏、模板下载 API 调用、按钮状态管理）

### Bug 修复记录：CSV import_csv 对 search_type=date 的处理缺失

- **Bug**：`import_csv()` (inputer.py) 对 `search_type="date"` 的处理完全错误——第 116 行 `pages = int(search_value) if search_type in ["pages", "all"] else 1`，当 `search_type="date"` 时走 else 分支，`search_value`（日期）被丢弃，硬编码为 `pages=1`，实际只拉了 1 页视频而非按日期过滤
- **根因**：`import_csv()` 没有根据 `search_type` 分发到对应的 `SearchService` 方法。`SearchService` 已有 `get_author_videos_before_date()` 方法可用，但从未被调用
- **发现方式**：全链路数据流审查（systematic-debugging），逐层追踪 CSV 上传 → 解析 → 服务调用的完整路径
- **修复**：在 `import_csv()` 中根据 `search_type` 分发：
  - `pages` / `all` → 调用 `get_author_videos(source_author_id, pages=N)`（行为不变）
  - `date` → 调用 `get_author_videos_before_date(source_author_id, before_date)`（新增）
- **测试**：`test_csv导入search_type分发逻辑.py`（7个验收点，TDD 红绿流程验证）
  - 3 个 date 类型测试（RED 阶段全部失败，证明 BUG 存在）
  - 2 个 pages 类型测试（确认现有行为不受影响）
  - 1 个 all 类型测试
  - 1 个响应结构测试
