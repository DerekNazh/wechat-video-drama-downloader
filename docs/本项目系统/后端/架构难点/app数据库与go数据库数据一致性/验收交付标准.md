# 双数据库一致性架构 - 验收交付标准

> **适用对象：** AI 自动化验收
> **验证时机：** 代码改动后立即验证
> **失败处理：** 自动修复后重新验证

---

## 验收流程

```
代码改动
    │
    ▼
触发验收项（按文件匹配）
    │
    ├─► 检测方法执行
    │       │
    │       ├─► 通过 → 继续
    │       │
    │       └─► 失败 → 执行修复策略 → 重新检测
    │                               │
    │                               ├─► 通过 → 继续
    │                               │
    │                               └─► 失败 → 报告异常，请求人工介入
    │
    ▼
验收完成
```

## 八、AI 执行指南

### 验收触发规则

1. **文件改动后**：检查改动文件是否匹配触发条件
2. **匹配成功**：执行对应的检测方法
3. **检测失败**：执行修复策略，重新检测
4. **修复失败**：报告异常，请求人工介入

### 自动修复原则

1. **测试失败**：分析失败原因，修复测试或修复代码
2. **导入失败**：检查模块导出，修复循环依赖
3. **启动失败**：检查初始化顺序，修复依赖问题

### 人工介入条件

1. 连续 3 次自动修复失败
2. 需要修改数据库结构
3. 需要修改外部依赖
4. 需要修改业务逻辑决策
