# 首页搜索作者入口重设计

## 背景

当前首页搜索框是假的 `<div>`，点击后跳转到搜索/导入页。随着作者数量增多，用户更需要**快速查找已有作者**而非重复添加。"搜索添加作者"的混合定位需要拆开。

## 方案

**搜索框 + `+` 按钮**

首页搜索框变成真正的 `<input>`，输入文字实时模糊过滤已有作者，浮出下拉卡片列表。`+` 圆形按钮紧贴搜索框右侧，点击跳转到现有的搜索/导入页（保持不变）。

```
┌─────────────────────────────┐ ┌──┐
│ 🔍 搜索作者...              │ │ +│
└─────────────────────────────┘ └──┘
  ┌───────────────────────────┐
  │ 🧑 张三  12视频  ▸        │
  │ 🧑 张三丰  5视频  ▸       │
  │ 🧑 李四  8视频  ▸         │
  └───────────────────────────┘
```

## 组件

### 1. 搜索输入框

- 将现有 `div.search-box.large` 替换为真正的 `<input id="homeSearchInput">`
- 占位符文字改为「搜索作者...」
- 输入事件 300ms 防抖，调用 `_allAuthors` 本地过滤（不请求后端）
- ESC 或点击外部关闭下拉

### 2. 下拉结果面板

- 绝对定位，贴在搜索框下方
- 最大高度 320px，内容溢出滚动
- 每条结果：头像(32px) + 作者名 + 视频数，点击跳转作者详情
- 无匹配时显示「未找到匹配的作者」灰色提示
- 空输入时隐藏下拉

### 3. `+` 添加按钮

- 紧贴搜索框右侧，蓝底白字圆形按钮
- 点击调用 `openSearchPage()`（跳转到现有搜索/导入页）
- hover 微微放大 + tooltip「添加作者」
- 与搜索框视觉一体，用 `border-radius` 右侧无缝连接

### 4. 筛选/视图控件

- 现有的筛选标签（全部/下载中/空闲）和视图切换按钮保持不变
- 位置不变，仍在搜索框下方那行

## 改动范围

| 文件 | 改动 |
|------|------|
| `static/index.html` (86-120) | 首页工具栏：搜索框改 `<input>`，加 `+` 按钮，加下拉面板容器 |
| `static/css/styles.css` | 搜索框 input 样式、下拉面板样式、`+` 按钮样式 |
| `static/js/component/homeSearchFilter.js` | **新文件**：防抖过滤 + 下拉渲染 + 键盘/点击交互 |
| `static/js/component/authorGridRender.js` | 复用作者数据源 `_allAuthors` |

**不改动**：搜索/导入页（`#pageSearch`）、导入卡片、作者详情页。

## 交互细节

1. 页面加载 → 搜索框显示占位符，下拉隐藏
2. 用户输入「张」→ 300ms 后过滤 `_allAuthors`，下拉显示匹配项
3. 点击「张三」→ 调用 `openAuthorDetail('张三')`，跳转作者详情
4. 点击 `+` → 调用 `openSearchPage()`，跳转搜索/导入页
5. 按 ESC → 清空输入，关闭下拉
6. 点击空白区域 → 关闭下拉

## 测试校验点

- 输入文字后下拉出现，内容匹配作者名
- 无匹配时显示空状态提示
- 点击结果跳转正确
- `+` 按钮跳转搜索页
- ESC 关闭下拉
- 点击外部关闭下拉
- 防抖 300ms 不过滤每次按键
