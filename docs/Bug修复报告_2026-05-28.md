# Bug 修复报告 - 2026-05-28

> **状态**: 已修复
> **修复日期**: 2026-05-28
> **范围**: 视频列表 UI/UX 大 Bug 修复（含卡片模式专项修复）

## Bug 1: SSE task_completed 回调崩溃（严重）

### 错误点

- `authorDetail.js:179-182` 引用 `document.getElementById("authorDownloaded")` / `authorPending` / `authorTotal`
- HTML 中不存在这些元素，导致 `null.textContent` 抛异常
- 异常中断 `handleSSETaskCompleted` 函数，后续的 `updateSingleVideoCompleted` 和 `_activeTasks` 清理均不执行

### 修复

- 将直接 DOM 操作替换为调用已有的 `updateAuthorStatsByType()` 函数
- 该函数内部已有 `getElementById` + 可选链保护，不会崩溃

## Bug 2: download_path 未转义导致 HTML 属性破坏（严重）

### 错误点

- `videoListRender.js:205,234` 模板中 `data-path="${video.download_path}"` 未做 HTML 转义
- Windows 路径含 `"` 或特殊字符时破坏 HTML 属性解析

### 修复

- 新增 `escapeAttr()` 函数（转义 `& " ' < >`）
- 模板中 `data-path` 改用 `escapeAttr(video.download_path)`

## Bug 3: 视频完成后仍留在"待下载"筛选视图

### 错误点

- `updateSingleVideoCompleted` 只检查 `currentFilter === 'downloading'` 来移除行
- 不检查 `pending` 筛选器，导致已完成视频带"已下载"徽章出现在"待下载"视图

### 修复

- 条件改为 `currentFilter === 'downloading' || currentFilter === 'pending'`

## Bug 4: 错误徽章出现在不匹配的筛选视图

### 错误点

- `updateSingleVideoError` 只处理 `downloading` 筛选器
- 失败视频带"下载失败"徽章出现在"已下载"筛选下

### 修复

- 条件改为 `currentFilter === 'downloading' || currentFilter === 'downloaded'`

## Bug 5: "全选"选中所有视频而非当前筛选结果

### 错误点

- `selectAllVideos()` 忽略筛选标签、搜索条件、日期范围
- 用户只看到 10 条却选中了 500 条

### 修复

- 使用 `getFilteredVideos()` 获取与当前显示一致的筛选结果
- 只选中筛选后的视频

## Bug 6: 空状态清除视图模式类

### 错误点

- `renderVideoList` 空状态路径 `el.className = 'video-list no-animation'` 剥离了 `grid-view` / `compact-view`
- 恢复数据后列表强制回到列表视图

### 修复

- 空状态路径保留当前视图模式类：`el.className = 'video-list no-animation ' + viewClass`

## Bug 7: 下载徽章 80px 宽度裁剪统计信息

### 错误点

- `.video-status-badge` 基础样式 `max-width: 80px; overflow: hidden`
- 下载中状态有三行内容（百分比、进度条、大小/速度），被裁剪不可读

### 修复

- 基础样式移除 `overflow: hidden`
- `.video-status-badge.downloading` 宽度改为 `110px`，添加 `white-space: normal; overflow: visible`

## Bug 8: ReactiveRenderer 慢路径缺少下载统计

### 错误点

- `ReactiveRenderer._updateProgressUI` 慢速路径 innerHTML 只含百分比和进度条
- 缺少 `download-stats`（文件大小/速度），用户看到残缺的下载状态

### 修复

- 补全 `download-stats` HTML，与 `updateVideoStatusIncremental` 保持一致

## Bug 9: 危险按钮悬停变蓝色

### 错误点

- `.btn-danger:hover` 使用 `var(--accent-hover)`（蓝色），破坏性操作悬停变蓝误导用户

### 修复

- 改为 `#b02a2d`（danger 色的深色变体）

## 卡片模式(grid-view)专项修复

## Bug A: 下载中状态框定位错误 — 覆盖信息区而非缩略图

### 错误点

- CSS `.video-status-badge.downloading` 在 grid-view 中设置 `position: absolute; bottom: 0; left: 0; right: 0;`
- 相对于 `.video-row`（flex-direction: column, position: relative）定位
- `bottom: 0` 让下载框贴在 row 底部——即 `.video-info` 区域而非缩略图区域

### 后果

1. 下载框遮挡视频标题和元数据
2. 渐变背景叠在浅色信息区上视觉混乱
3. 进度信息本应叠在缩略图上方作为视觉反馈，现在叠在文字上毫无意义

### 修复

- 改为 `top: 0; height: 112px; bottom: auto;` 精确覆盖缩略图区域
- `justify-content: flex-end` 让内容（百分比+进度条+统计）贴底
- 移除 `border-radius`（由 row 的 `overflow: hidden` 自动裁剪）

## Bug B: 播放按钮定位在卡片中心而非缩略图中心

### 错误点

- `.video-open-btn` 在 grid-view 中 `top: 50%; left: 50%; transform: translate(-50%, -50%)`
- 相对于 `.video-row` 的中心——但 row 包含缩略图(112px) + 信息区(~50px)
- 按钮出现在卡片整体中心而非缩略图中心，视觉上偏下

### 修复

- 改为 `top: 56px`（112px / 2），精确对齐缩略图中心
- hover 状态保留 `translate(-50%, -50%) scale(1.1)`
