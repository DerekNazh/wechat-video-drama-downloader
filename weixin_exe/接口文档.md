# 视频号下载器 API 接口文档

**基础地址**：`http://127.0.0.1:2022`

**通信方式**：HTTP / JSON

**统一响应格式**：
```json
{
  "code": 0,
  "msg": "ok",
  "data": {}
}
```

| code | 含义 |
|------|------|
| 0 | 成功 |
| 400 | 参数错误 |
| 404 | 资源不存在 |
| 409 | 冲突（如任务已存在） |
| 500 | 服务器内部错误 |

---

## 一、服务状态接口

### 1. GET /api/status

**用途**：检查 API 服务是否存活、获取版本信息和 channels 功能状态。（go源码有bug 暂不可用）

**请求示例**：
```
GET http://127.0.0.1:2022/api/status
```

**成功响应**：
```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "version": "260330",
    "channels": {
      "available": true
    }
  }
}
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| version | string | API 版本号（日期格式：年月日） |
| channels.available | bool | 视频号功能是否可用（依赖证书安装状态） |

### 重要：普通视频接口与直播回放接口的数据隔离

**实测验证结论**（2026-04-15）：

1. **接口完全独立**：`/api/channels/contact/feed/list`（普通视频）和 `/api/channels/live/replay/list`（直播回放）是两个完全独立的接口
   - 数据源不同（微信后端不同的存储）
   - **ID 完全不重叠**：同一作者的普通视频 ID 和回放 ID 是两个互不相关的集合

2. **返回结构相似但数据不同**：
   - 普通视频的 `videoPlayLen` 通常较短（100-400秒）
   - 直播回放的 `videoPlayLen` 通常很长（5000秒以上）
   - 可通过 `videoPlayLen` 区分视频类型

3. **作者头像可用**：两个接口返回的每个视频/回放对象中，`contact.headUrl` 字段包含作者头像 URL
   - 格式：`https://wx.qlogo.cn/finderhead/ver_1/.../132`
   - 同一作者在不同视频中的头像 URL 一致

4. **时间排序**：API 返回的数据不保证按时间排序，客户端需自行按 `createtime` 排序（建议倒序，最新在前）

### 10. POST /api/task/create

**用途**：创建单个视频下载任务。

**请求体**：

```json
{
  "id": "视频ID",
  "nonce_id": "nonce_xxxxx",
  "url": "https://finder.video.qq.com/...",
  "title": "视频标题",
  "filename": "下载后的文件名（不含后缀）",
  "key": 1234567890,
  "spec": "xWT111",
  "suffix": ".mp4"
}
```

**请求示例**：
```bash
curl -X POST http://127.0.0.1:2022/api/task/create \
  -H "Content-Type: application/json" \
  -d '{
    "id": "4321567890123456789",
    "url": "https://finder.video.qq.com/...",
    "title": "残局505关",
    "filename": "残局505关",
    "key": 1234567890,
    "spec": "xWT111",
    "suffix": ".mp4"
  }'
```

**成功响应**：
```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "id": "fraubhofMRgtllLyzQk6r"
  }
}
```

**重要：任务去重行为**

- 通过 `id + spec + suffix` 组合判断任务是否已存在
- 若任务已存在，API 返回 `{"code": 409, "msg": "已存在该下载任务"}`，这是正常行为，不是错误
- `batch_create_tasks()` API 会自动跳过已存在的任务，只返回新创建的任务 ID 列表

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 视频 ID |
| nonce_id | string | 否 | 视频 Nonce ID |
| url | string | 是 | 视频直链（media.url + media.urlToken） |
| title | string | 否 | 视频标题（用于记录） |
| filename | string | 否 | 下载文件名（不含后缀和规格标识） |
| key | int | 否 | 解密密钥（media.decodeKey 转数字） |
| spec | string | 否 | 画质规格标识 |
| suffix | string | 否 | 文件后缀（.mp4/.mp3/.zip） |

curl -X POST http://127.0.0.1:2022/api/task/create_batch \

## 八、自动监控方案

### 基于接口的自动下载流程

```
1. 搜索作者
   GET /api/channels/contact/search?keyword=作者名
   → 获取 contact.username

2. 获取视频列表（第一页）
   GET /api/channels/contact/feed/list?username=xxx
   → 获取 object[].id、object[].createtime

3. 遍历所有页（如果有）
   获取 lastBuffer，用 next_marker 继续请求

4. 过滤新视频
   对比本地已记录的 max(createtime)，只下载新视频

5. 批量创建下载任务
   POST /api/task/create_batch
   → 获取任务 ID 列表

6. 定时轮询
   每隔一定时间（如 30 分钟）重复步骤 2-5
```

### 自动监控配置建议

```yaml
# config.yaml
download:
  dir: "D:/微信视频"
  filenameTemplate: "{{author}}/{{created_at}}_{{title}}_{{spec}}"
  pauseWhenDownload: false
  playDoneAudio: true
```

## 十、使用示例

### 搜索并下载指定作者的所有视频

```python
import requests
import time

API_BASE = "http://127.0.0.1:2022"

def search_author(keyword):
    """搜索作者"""
    resp = requests.get(f"{API_BASE}/api/channels/contact/search", params={"keyword": keyword})
    data = resp.json()
    if data["code"] == 0 and data["data"]["infoList"]:
        return data["data"]["infoList"][0]["contact"]
    return None

def get_author_videos(username):
    """获取作者所有视频"""
    videos = []
    marker = ""
    while True:
        params = {"username": username}
        if marker:
            params["next_marker"] = marker
        resp = requests.get(f"{API_BASE}/api/channels/contact/feed/list", params=params)
        data = resp.json()
        if data["code"] != 0:
            break
        videos.extend(data["data"]["object"])
        if data["data"]["continueFlag"] == 0:
            break
        marker = data["data"]["lastBuffer"]
    return videos

def download_videos(videos):
    """批量下载视频"""
    feeds = []
    for v in videos:
        media = v["objectDesc"]["media"][0]
        feeds.append({
            "id": v["id"],
            "nonce_id": v.get("objectNonceId", ""),
            "url": media["url"] + media.get("urlToken", ""),
            "title": v["objectDesc"]["description"],
            "key": int(media.get("decodeKey", "0")),
            "spec": media["spec"][-1] if media["spec"] else "",
            "suffix": ".mp4"
        })
    resp = requests.post(f"{API_BASE}/api/task/create_batch", json={"feeds": feeds})
    return resp.json()

# 执行
author = search_author("象棋大师")
if author:
    videos = get_author_videos(author["username"])
    print(f"找到 {len(videos)} 个视频")
    result = download_videos(videos)
    print(f"创建了 {len(result['data']['ids'])} 个下载任务")
```

## 十一、E2E 测试覆盖与生产保证

| 测试层 | 位置 | 验证内容 |

| **端到端测试** | `tests/端到端测试/` | API 层完整链路（HTTP 接口→监控器→真实后端服务→微信数据） |

### 生产保证总结

**当全部 E2E 测试通过时，以下生产场景可保证正常运行：**

1. **用户启动监控** → 点击按钮 → 监控器启动 → 状态正确显示"运行中"
2. **定时检查回放** → 调度器按时触发 → 调用微信 API 获取回放列表 → 识别新回放
3. **创建下载任务** → 新回放自动创建下载任务 → 后端开始下载
4. **状态实时同步** → 前端轮询 `/api/status` → 服务状态/微信连接/监控状态/今日新增全部正确
5. **作者列表正确** → 所有预置作者正确加载 → 头像 URL 已填充
6. **日志完整记录** → 检查过程写入日志 → 前端可查看
7. **用户停止监控** → 点击停止 → 调度器停止 → 资源释放 → 状态显示"已停止"
8. **按钮幂等性** → 重复点击启动/停止不崩溃

### 不在保证范围内

以下场景 E2E 测试**未覆盖**，需要其他测试或人工验证：

| 场景 | 原因 | 覆盖方式 |
|------|------|----------|
| 服务启动/停止（`/api/service/start`, `/api/service/stop`） | 有进程管理副作用 | 手动验证 |
| 微信断连后重连 | 需要模拟微信断连 | 手动验证 |
| 多用户并发访问 | 需要并发测试工具 | 压力测试 |
| 前端 UI 渲染 | 需要 Playwright/Selenium | UI E2E 测试 |
| 网络超时/重试 | 需要 mock 网络异常 | 单元测试 |
| 数据库损坏恢复 | 需要构造损坏文件 | 手动验证 |
| 长时间运行内存泄漏 | 需要长时间监控 | 性能测试 |
