# config.yaml 配置指南

本文档详细说明 `wx_video_download.exe` 的配置文件 `config.yaml` 各项参数的含义和使用方法。

---

## 配置文件位置

配置文件 `config.yaml` 需要与 `wx_video_download.exe` 放在同一目录下。程序启动时会自动读取该文件，如果不存在则使用默认值。

---

## 完整配置示例

```yaml
# 调试配置
debug:
  error: false

# PageSpy 调试功能
pagespy:
  enabled: false

# 下载配置
download:
  dir: "%UserDownloads%"
  filenameTemplate: "{{filename}}_{{spec}}"
  defaultHighest: false
  pauseWhenDownload: false
  playDoneAudio: true
  frontend: false
  forceCheckAllFeeds: false

# API 服务配置
api:
  protocol: "http"
  hostname: "127.0.0.1"
  port: 2022

# 代理服务配置
proxy:
  system: true
  hostname: "127.0.0.1"
  port: 2023
  skipInstallRootCert: false

# 证书配置（可选）
cert:
  file: ""
  key: ""
  name: "Echo"

# 视频号配置
channels:
  disableLocationToHome: false

# 公众号配置
mp:
  disabled: true

# Cloudflare 远程服务配置（可选）
cloudflare:
  accountId: ""
  apiToken: ""
  refreshToken: ""
  adminToken: ""
  workerName: "mp-rss-api"
  d1Name: "mp-rss-db"
```

## 配置项详细说明

### 一、下载配置 (download)

控制视频下载的核心行为。

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `dir` | string | `%UserDownloads%` | 下载目录路径 |
| `filenameTemplate` | string | `{{filename}}_{{spec}}` | 下载文件名模板 |
| `defaultHighest` | bool | false | **已失效** - 点击下载图标时是否下载原始视频 |
| `pauseWhenDownload` | bool | false | 点击下载时是否暂停播放 |
| `playDoneAudio` | bool | true | 下载完成时是否播放提示音 |
| `frontend` | bool | false | 是否通过前端解密下载（不调用后台下载能力）|
| `forceCheckAllFeeds` | bool | false | 批量下载时是否强制检查所有视频 |

#### 1. 下载目录 (download.dir)

支持以下特殊路径变量：

| 变量 | 说明 | 示例 |
|------|------|------|
| `%UserDownloads%` | 用户系统下载目录 | `C:\Users\用户名\Downloads` |
| 相对路径 | 相对于 exe 所在目录 | `./downloads`、`../videos` |
| 绝对路径 | 完整的文件路径 | `D:\Videos\微信视频` |

**示例：**

```yaml
download:
  dir: "%UserDownloads%"              # 系统下载目录
  # dir: "./downloads"                # 程序目录下的 downloads 文件夹
  # dir: "D:/微信视频下载"             # 指定绝对路径
  # dir: "../downloads"               # 上级目录的 downloads 文件夹
```

#### 2. 文件名模板 (download.filenameTemplate)

支持以下变量替换：

| 变量 | 说明 | 示例值 |
|------|------|--------|
| `{{filename}}` | 视频描述（默认文件名）| `残局505关欢迎高手挑战` |
| `{{id}}` | 视频 ID | `1334708551639` |
| `{{title}}` | 视频标题（同 filename）| `残局505关欢迎高手挑战` |
| `{{spec}}` | 视频规格标识 | `xWT111`、`xWT98` |
| `{{created_at}}` | 视频发布时间戳 | `1712966400` |
| `{{download_at}}` | 下载时间戳 | `1713052800` |
| `{{author}}` | 作者昵称 | `象棋教学` |

**常用模板示例：**

```yaml
download:
  # 基础格式：描述_规格
  filenameTemplate: "{{filename}}_{{spec}}"

  # 按作者分类
  filenameTemplate: "{{author}}/{{filename}}_{{spec}}"

  # 按日期+作者+标题
  filenameTemplate: "{{created_at}}_{{author}}_{{title}}_{{spec}}"

  # 按作者目录+日期标题
  filenameTemplate: "作者视频/{{author}}/{{created_at}}_{{title}}_{{spec}}"
```

**注意：** 当模板中包含路径分隔符 `/` 时，程序会自动创建相应的子目录。

### 二、API 服务配置 (api)

本地 API 服务的网络配置，用于提供视频下载、任务管理等接口。

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `protocol` | string | `http` | API 服务协议 |
| `hostname` | string | `127.0.0.1` | API 服务主机名 |
| `port` | int | 2022 | API 服务端口 |

**示例：**

```yaml
api:
  protocol: "http"
  hostname: "127.0.0.1"
  port: 2022
```

API 启动后可通过以下地址访问：

- 状态检查：`http://127.0.0.1:2022/api/status`
- RSS 订阅：`http://127.0.0.1:2022/rss/channels?username=xxx`

---

### 三、代理服务配置 (proxy)

MITM 代理服务配置，用于拦截微信视频号的网络请求。

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `system` | bool | true | 是否设置系统代理 |
| `hostname` | string | `127.0.0.1` | 代理服务主机名 |
| `port` | int | 2023 | 代理服务端口 |
| `skipInstallRootCert` | bool | false | 是否跳过安装根证书 |

**注意事项：**

1. 首次运行需要**管理员权限**，以便安装 HTTPS 解密证书
2. 如果已安装过 mitmproxy 等工具的证书，程序会自动复用
3. 设置 `skipInstallRootCert: true` 时，需要手动信任证书

**示例：**

```yaml
proxy:
  system: true                # 自动设置系统代理
  hostname: "127.0.0.1"
  port: 2023
  skipInstallRootCert: false  # 首次运行建议设为 false
```

---

### 四、证书配置 (cert)

可选配置，用于自定义 HTTPS 解密证书。

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `file` | string | 空 | 证书文件绝对路径 |
| `key` | string | 空 | 私钥文件绝对路径 |
| `name` | string | `Echo` | 证书名称 |

**示例：**

```yaml
cert:
  file: "D:/certs/mycert.pem"
  key: "D:/certs/mykey.pem"
  name: "MyCert"
```

---

### 五、视频号配置 (channels)

视频号相关功能配置。

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `disableLocationToHome` | bool | false | 是否禁止从视频详情页重定向到首页 |

**说明：** 微信视频号默认会从详情页重定向到首页，设置 `true` 可禁止此行为。

---

### 六、调试配置 (debug)

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `error` | bool | false | 是否全局捕获前端错误并弹窗显示 |

---

### 七、PageSpy 配置 (pagespy)

用于远程调试微信 PC 端页面。

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | bool | false | 是否启用 PageSpy |
| `protocol` | string | `https` | PageSpy 调试协议 |
| `api` | string | `debug.weixin.qq.com` | PageSpy API 地址 |

---

### 八、公众号配置 (mp)

公众号文章下载相关配置。

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `disabled` | bool | true | 是否禁用公众号本地服务 |
| `refreshToken` | string | 空 | 远端服务刷新凭证 |
| `refreshSkipMinutes` | int | 20 | 刷新跳过时间（分钟）|

---

### 九、Cloudflare 配置 (cloudflare)

用于部署远程服务的 Cloudflare 配置（高级用法）。

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `accountId` | string | 空 | Cloudflare 帐号 ID |
| `apiToken` | string | 空 | Worker 认证 Token |
| `refreshToken` | string | 空 | 凭证刷新 Token |
| `adminToken` | string | 空 | 管理员接口凭证 |
| `workerName` | string | `mp-rss-api` | Worker 名称 |
| `d1Name` | string | `mp-rss-db` | D1 数据库名称 |

## 常见配置场景

### 场景一：基础使用（默认配置）

适合大多数用户，直接使用默认配置即可：

```yaml
download:
  dir: "%UserDownloads%"
  filenameTemplate: "{{filename}}_{{spec}}"
  playDoneAudio: true

api:
  port: 2022

proxy:
  system: true
  port: 2023
```

### 场景二：按作者分类存储

自动按作者创建文件夹，便于整理：

```yaml
download:
  dir: "D:/微信视频"
  filenameTemplate: "{{author}}/{{created_at}}_{{title}}_{{spec}}"
```

### 场景三：自动化监控下载

配合 API 接口实现自动下载：

```yaml
download:
  dir: "./downloads"
  filenameTemplate: "{{author}}/{{title}}_{{spec}}"
  pauseWhenDownload: false

api:
  hostname: "127.0.0.1"
  port: 2022
```

然后通过脚本调用 API：
- `GET /api/channels/contact/search?keyword=作者名` - 搜索作者
- `GET /api/channels/contact/feed/list?username=xxx` - 获取视频列表
- `POST /api/task/create_batch` - 批量创建下载任务

### 场景四：RSS 订阅

启用 RSS 功能，可通过 RSS 阅读器订阅作者更新：

```
http://127.0.0.1:2022/rss/channels?username=作者username
```

## 配置验证

程序启动时会自动验证配置文件。如果配置有误，会在日志中显示错误信息。

验证 API 是否正常：

```bash
curl http://127.0.0.1:2022/api/status
```

正常返回：

```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "version": "260330",
    "channels": {
      "available": true
    }
  }
}
```

---

## 注意事项

1. **管理员权限**：首次运行需要管理员权限安装证书
2. **端口冲突**：确保 2022 和 2023 端口未被占用
3. **路径格式**：Windows 路径可使用正斜杠 `/` 或双反斜杠 `\\`
4. **证书问题**：如遇到证书错误，可尝试删除旧证书重新安装
5. **编码问题**：配置文件使用 UTF-8 编码
